## ğŸ“– Theory: The Architecture Flow
Git works across four distinct areas. Understanding this flow is critical for mastering Git:
1. **Working Directory:** Where you modify files.
2. **Staging Area (Index):** Preparing files for a commit.
3. **Local Repo (.git):** Where snapshots are saved locally.
4. **Remote Repo (GitHub):** Where the code lives in the cloud.

# Phase 2: Basic Workflow (The Daily Cycle)

## ğŸ¯ Learning Objectives
* Connect local machines to GitHub.
* Master the `Add -> Commit -> Push` cycle.
* Understand the difference between `Pull` and `Fetch`.

---

## ğŸ“– Essential Commands
| Command | Description |
| :--- | :--- |
| `git clone <url>` | Copies a remote repository to your local machine. |
| `git status` | Shows which files are staged, unstaged, and untracked. |
| `git add <file>` | Moves changes from the Working Directory to the Staging Area. |
| `git commit -m "msg"` | Saves the staged snapshot to the Local Repository. |
| `git push` | Uploads local commits to GitHub. |
| `git fetch` | Downloads updates from GitHub but does not merge them. |
| `git pull` | Downloads and automatically merges updates (Fetch + Merge). |

## ğŸ“ The .gitignore File
Always create a `.gitignore` file to prevent sensitive data (like `.env`) or system junk (like `node_modules`) from being uploaded.

## ğŸ“– Key Concepts
### 1. Merging
Combines two branches while preserving the history of how they joined.
```bash
git checkout main
git merge feature-branch
2. Rebasing
"Re-writes" history by moving your feature branch to the tip of the main branch. It creates a cleaner, linear history.

3. Merge Conflicts
Occur when two people change the same line of code.

Solution: Open the file, choose the correct code, git add, and git commit.

# Phase 5: Undo & Troubleshoot (The Panic Button)

## ğŸ¯ Learning Objectives
* Learn how to safely undo mistakes.
* Understand the "Dangerous" vs. "Safe" undo commands.

---

## ğŸ“– Revert vs. Reset
| Command | Effect | Safety |
| :--- | :--- | :--- |
| `git revert` | Creates a NEW commit that does the opposite of a previous one. | **Safe** (Best for shared code) |
| `git reset --soft` | Undoes the commit but keeps your code changes. | **Medium** |
| `git reset --hard` | Deletes the commit and all code changes. | **Dangerous** (Data loss!) |

## ğŸ› ï¸ Common Errors & Solutions
* **"Updates were rejected":** You need to `git pull` before you can `git push`.
* **"Detached HEAD":** You checked out a commit instead of a branch. Run `git checkout main`.
